Welcome to Verboscript playground!

Here you will learn a couple of things about this language, feel free to toy as much as you want with the code samples and you'll see the result and the translated JavaScript below. Ready?

Lines and blanks

A program in Verboscript is no more than a bunch of sentences or statements separated by new lines. Only one statement per line is allowed. Verboscript is quite strict and will require you to pay some attention to unnecesary blanks. For instance, it does not allow trailing spaces and it requires to be blankspaces in both sides of the operators.

Statements

Those sentences altering the state of the program are called statements in opposition to those with no side effects refered as transparent expressions or simply expressions.

Assignment statements

The sentence to introduce a new object in the program is the assignation. There is no variable declaration in Verboscript, you only need to assign some value to a name. Use the nil symbol (<>) to introduce a name without content.

Definitions

A definition introduces a new function. When defining a function you can use words and supporting words to explain the purpose of the function. The first not skipped word (those starting by `) becomes the name of the function and the remaining ones become the parameters.

Calls

A call causes the body of a function to be executed. Verboscript identify a call by recognizing any sequence of identifiers, memory or expressions. The first defined symbol is considered the name of the function and the remaining defined ids or expressions, the parameters.

If there is no function name identified, the sentence is simply ignored. You can force to ignore some ids by prefixing them with `.
